Como instalar o plugin ?

Opção 1:

  1. 1. Feche o Eclipse se estiver aberto.
  2. 2. Baixe
  3. 3. Coloque o .jar dentro de {eclipse_folder}\dropins\
  4. 4. Abra o Eclipse.

Opção 2:

  1. 1. No menu do Eclipse, selecione Help -> Install New Software...
  2. 2. Na janela Install, no campo Work with, digite http://143.107.58.177/baduino/
  3. 3. Marque a ultima versão da Baduino e clique em Next
  4. 4. Siga os passos do instalador.

A partir de agora, é possível verificar as Associações Definição-Uso (ADUs) do programa, porém ainda não é possível coletar a cobertura de ADUs. Para saber como utilizar o plugin click aqui

E o que eu preciso para coletar a cobertura de DUAs ?

  1. Localização padrão: [projectDir]/jacocoagent.jar
  2. Localização padrão: [projectDir]/jaguar.jar
  3. As classes compiladas. Localização padrão: [projectDir]/target/classes
  4. As classes de teste compiladas. Localização padrão: [projectDir]/target/test-classes

E se meu projeto não usa a estrutura de diretórios padrão do maven ?

Se um dos arquivos (jacocoagent.jar e jaguar.jar) ou pastas (classes e classes de teste) não estiverem na localização padrão, você pode especificar o caminho manualmente, criando um arquivo na raiz do projeto chamado baduino.properties. Esse arquivo deve conter as seguintes informações:

  • jacoco.agent.jar=<localização do jacocoagent.jar>
  • jaguar.jar=<localização do jaguar.jar>
  • compiled.tests.dir=<localização dos pacotes de teste compilados>
  • compiled.classes.dir=<localização das classes compiladas>
  • project.dir=<Localização da raiz do projeto>

Obs: Todas as classes de testes devem terminar com *Test.class
